# 就活管理アプリ 開発記録

---

## 1. プロジェクト概要

### アプリ名
就活管理アプリ (Shukatsu App)

### 目的
就活における企業情報の整理・比較、スケジュール管理、Googleカレンダー連携を一元化する。

### URL
- 標準版（ポートフォリオ用）: https://shukatsu-app-chi.vercel.app
- カレンダー連携版（自分用）: https://shukatsu-app-calendar.vercel.app
- GitHub: https://github.com/kouta-funatsu/shukatsu-app

---

## 2. ブランチ戦略

| ブランチ | デプロイ先 | 内容 |
|---|---|---|
| `main` | shukatsu-app-chi.vercel.app | カレンダーなし・誰でもログイン可能・ポートフォリオ用 |
| `feature/calendar` | shukatsu-app-calendar.vercel.app | カレンダー・Gmail連携あり・テストユーザーのみ・自分用 |

GoogleカレンダーAPIはGoogleの審査を受けないと一般公開できない。審査にはプライバシーポリシー・動画・セキュリティ審査（有料）が必要なため、カレンダー連携機能は`feature/calendar`ブランチに分離している。

## 6. 発生したエラーと解決方法

### エラー1：`touch: No such file or directory`
**原因：** フォルダが存在しない状態でファイルを作ろうとした。  
**解決：** 先に`mkdir`でフォルダを作成してからファイルを作成。

### エラー2：Vercelデプロイエラー（環境変数なし）
**原因：** `.env.local`はローカルにしか存在せず、Vercelに環境変数が未設定だった。  
**解決：** Vercel → Settings → Environment Variables に追加。

### エラー3：Googleログイン後にlocalhostに戻る
**原因：** SupabaseのリダイレクトURLに本番URLが未設定だった。  
**解決：** Supabase → Authentication → URL Configuration で本番URLを追加。

### エラー4：`Cookies can only be modified in a Server Action`
**原因：** Server ComponentでCookieを直接setしようとした。  
**解決：** `setAll`の中を`try {} catch {}`で囲む。

### エラー5：`cannot drop constraint companies_pkey`
**原因：** company_accountsテーブルがcompaniesの主キーを参照していた。  
**解決：** company_accountsを削除してからcompaniesを編集し再作成。

### エラー6：`params is a Promise and must be unwrapped with await`
**原因：** Next.js 15からparamsがPromiseになった。  
**解決：** `params: Promise<{id: string}>`に変更して`await params`で展開。

### エラー7：Tailwindが効かない
**原因：** tailwind.config.tsが存在せず、Tailwind v4がインストールされていた。  
**解決：** v4はconfigファイル不要。globals.cssに`@import "tailwindcss"`を追加。

### エラー8：ログインするとVercelのURLに飛ぶ
**原因：** `NEXT_PUBLIC_SITE_URL`がVercelのURLに設定されていた。  
**解決：** `.env.local`に`NEXT_PUBLIC_SITE_URL=http://localhost:3000`を追加。

### エラー9：`createBrowserClient in non-browser runtimes`
**原因：** Server ComponentからcreateClientを呼び出していた。  
**解決：** Server ComponentではcreateServerClientを直接使うように修正。

### エラー10：ブランチマージでコンフリクト
**原因：** mainとfeature/calendarの両方でlogin/page.tsxを変更していた。  
**解決：** VSCodeのマージエディタで「入力側の変更を取り込む」を選択して手動解決。
